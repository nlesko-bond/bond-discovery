## Key audit findings behind these plans (evidence verified by direct read)

1. `/api/pages` POST/PATCH/DELETE have no auth (`app/api/pages/route.ts:24`, `app/api/pages/[slug]/route.ts:34,64`) → plan 001.
2. `public/bond-host/v1.js:219-247` never handles `BOND_GTM_EVENT` — Bond's documented checkout conversion pattern (Intercom 11139229/12580240) is silently dropped in kit-mounted checkouts → plan 003.
3. `components/analytics/GoogleTagManager.tsx` fires 11 event types inside the discovery iframe; operator wants a minimal contract → plan 004.
4. `lib/cache.ts` + `app/api/cron/warm-discovery/route.ts` had zero tests; cron auth is skipped when `CRON_SECRET` unset (`route.ts:51-55`) → plan 002.
5. `lib/supabase.ts:3-8` hardcodes production URL + anon key as fallbacks — every misconfigured environment silently becomes production → plan 005. (Anon key also lives in git history; rotation is an operator task.)
6. Six dead `" 2.ts"` duplicates in `lib/host-shell/` → plan 006.
7. Embed kit (`public/embed-kit/v1.js`, 1,683 lines + routes + admin section) duplicates the host kit → plan 007.
8. New pages are never warmed on create (cold path up to 15 min); invalidation misses `eventHorizonMonths`/`bondEnv` changes; programs cache lacks SWR → plan 006.

## Findings considered and rejected (do not re-audit)

- **Per-org Bond API quota tracking**: real risk only if shared-key orgs grow; current scope-grouping already minimizes calls. Revisit if `bondApi.rateLimitHits` trends up in cron responses. Deferred, noted in plan 006.
- **Moving partner GTM out of the discovery iframe to parent-page emission**: architecturally cleaner (cross-domain attribution) but a breaking change to partner setups; deferred as an explicit follow-up in plan 004.
- **`buildRegistrationUrl()` string-concat fallback** (`lib/utils.ts:328-360`): fragile in theory, but it only triggers when `new URL()` throws on Bond-supplied linkSEO — low real-world incidence; not worth a plan. Tighten opportunistically if touched.
- **10-second pageview debounce in `lib/analytics.ts:30`**: internal analytics nuance, no partner impact.
- **Dropping `features.embed*` columns from Supabase**: intentionally NOT planned — leave dead config data; deleting columns is irreversible and buys nothing.
- **GTM dedupe Set silently merging identical Bond/partner container IDs** (`GoogleTagManager.tsx:49`): correct behavior (loading the same container twice is the bug); no action.
